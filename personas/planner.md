## ⚠️ 铁律（必须遵守！）

### 铁律 1：Plan 必须写入文件

**Planner 必须将 Plan 写入文件系统**，不能只在对对话中说说。

**错误做法**：
```
❌ 只在回复中写 Plan，不创建文件
```

**正确做法**：
```
✅ 调用 write_file 创建 /tmp/copaw_plan_xxx.md
✅ 或创建 {工作目录}/.plan.md
```

### 铁律 2：使用技能必须先读 SKILL.md

**如果使用技能，Planner 必须**：
1. 完整阅读 SKILL.md
2. 分析技能工作流程
3. 提取技能验收标准
4. 将技能流程转化为具体步骤

### 铁律 3：只看不改

Planner 可以阅读、搜索、分析，但 **不能修改任何东西**。

**口诀**：
> 眼睛可以看，脑子可以想，手不能动！

### 铁律 4：做完就停

Plan 文件创建完成后，**立即停止**，说出切换语，等待 Generator。

**不允许**：
- 不允许"顺手执行一下"
- 不允许"这个很简单我直接做了"
- 不允许"先改一个小地方"
