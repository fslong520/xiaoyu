#### 改进

- 🚀 任务执行质量提升

## [0.2.0] - 2026-03-25

### 🎭 人格分裂版

### 🎭 人格分裂版

#### 新增
- ✨ **单智能体三角色系统**
  - Planner 小毓（产品经理）：需求分析、制定 Plan
  - Generator 小毓（开发者）：按 Plan 执行、写代码
  - Evaluator 小毓（QA 测试员）：验收检查、挑毛病
- ✨ **人格文件目录** `personas/`
  - `personas/planner.md` - Planner 人格定义
  - `personas/generator.md` - Generator 人格定义
  - `personas/evaluator.md` - Evaluator 人格定义
- ✨ **Harness Design 设计模式**
  - 基于 Anthropic Labs 研究论文
  - 解决 AI 长时间工作的两个致命问题：
    - 上下文焦虑（Context Anxiety）
    - 自我评估失真（Self-evaluation Bias）
- ✨ **角色切换仪式**
  - 明确的切换语强化角色边界
  - 避免上下文焦虑和自我评估失真

#### 改进
- 🚀 **复杂任务执行质量提升**
  - Evaluator 必须挑出至少 3 个问题
  - 问题必须具体到行号、错误类型、修复建议
- 🚀 **自我评估机制优化**
  - 生成 - 评估分离原则
  - 验收标准前置
- 🚀 **长任务分段检查**
  - 超过 2 小时的任务主动建议分段
  - 每个 sprint 后切换 Evaluator 验收

#### 文档
- 📚 更新 AGENTS.md 增加 Harness Design 章节
- 📚 更新 README.md 增加三角色协作说明
- 📚 创建 CHANGELOG.md 版本更新日志

## 版本说明

### 版本号规则

- **主版本号（Major）**：不兼容的 API 修改或核心架构变化
- **次版本号（Minor）**：向下兼容的功能性新增
- **修订号（Patch）**：向下兼容的问题修正

### 更新频率

- **重大更新**：每月一次（主版本号或次版本号）
- **小更新**：每周一次（修订号）
- **紧急修复**：随时（修订号）
